# 环境准备与工具链

> 这一章把开发环境搭好,并跑通你的第一个 LLM 请求。我们会顺手封装一个最小的 `chat()` 统一抽象——本书后续所有代码都会复用它,方便在不同模型厂商之间切换。

> 学习目标
> - 装好 Node.js 和 Python 双环境,并理解为什么两个都要
> - 学会用环境变量安全管理 API Key(绝不硬编码、绝不提交)
> - 跑通第一个 "Hello LLM" 请求,TypeScript + Python 双语
> - 搭好一个最小的 `chat()` 抽象,作为全书代码的基座
> - 知道常见环境坑长什么样,出问题能自查

前置知识:命令行基本操作、装过 npm 包。

## 为什么是双环境

本书代码双语:TypeScript 是你的主场,Python 是 Agent 生态的通用语。所以两个运行时都要装。

- **Node.js**:跑 TS/JS 代码。前端项目、BFF、流式接口用它最顺。
- **Python**:跑 Python 代码。大量 Agent 库、向量库、评测工具只有 Python 版,绕不开。

平时你可以只用主语言写,但**至少要能把另一种语言的示例跑起来**,否则会错过半个生态。

## 安装 Node.js 与 Python

### Node.js 18+ / 20+

到 [nodejs.org](https://nodejs.org) 下载 LTS 版本(20.x 最稳),或用版本管理器(推荐):

```bash
# 用 nvm(推荐,方便切换版本)
nvm install 20
nvm use 20

# 验证
node -v   # 应 >= v18,推荐 v20
npm -v
```

> 为什么至少 18?现代 Node SDK 用到了原生 `fetch`、ESM、顶层 `await` 等特性。低于 18 可能跑不起来。

### Python 3.10+

到 [python.org](https://python.org) 下载,或用 `pyenv` 管理版本:

```bash
# 用 pyenv(推荐)
pyenv install 3.12
pyenv global 3.12

# 验证
python3 --version   # 应 >= 3.10
```

> 为什么至少 3.10?很多库用到了 3.10+ 的类型注解写法和 `match` 语句,本书 Python 示例也按 3.10+ 编写。

## 包管理

### TypeScript:pnpm 或 npm

两个都行,本书示例用 `pnpm`(快、省磁盘),不习惯就用 `npm`,命令几乎一样。

```bash
# 初始化一个项目
mkdir my-agent && cd my-agent
pnpm init          # 或 npm init -y

# 用 ESM(在 package.json 里加一行 "type": "module")
# 装 TS 工具链
pnpm add -D typescript tsx @types/node
# tsx 让你不用先编译就能直接跑 .ts 文件:pnpm tsx xxx.ts
```

### Python:venv + pip,或 uv

**永远用虚拟环境**,不要往系统 Python 里乱装包。两种主流方式:

```bash
# 方式一:标准库 venv + pip(无需额外安装)
python3 -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install --upgrade pip

# 方式二:uv(更快的新一代工具,推荐尝试)
# 安装 uv: curl -LsSf https://astral.sh/uv/install.sh | sh
uv venv
source .venv/bin/activate
# uv pip install 用法和 pip 一致,但快很多
```

> 看到命令行提示符前面出现 `(.venv)`,说明虚拟环境已激活。之后 `pip install` 装的包都只在这个环境里,干净隔离。

## 获取并安全管理 API Key

要调模型,先去厂商控制台拿 API Key:

- **Anthropic(Claude)**:[console.anthropic.com](https://console.anthropic.com) → 注册 → 充值/开通计费 → 创建 API Key。
- **OpenAI**:[platform.openai.com](https://platform.openai.com) → 注册 → 开通计费 → 创建 API Key。

> ⚠️ **两家都需要先开通计费(绑卡或充值)才能成功调用。** 很多人第一次失败就是因为账户里没钱或没开通,这点后面"常见坑"会再强调。

### 密钥安全:三条铁律

API Key 等于你的钱包。泄露了,别人能用你的额度,甚至刷爆你的账单。记住三条:

1. **绝不硬编码进代码。** 不要写 `const key = "sk-ant-xxx"`。
2. **用环境变量 + `.env` 文件。** 把密钥放进 `.env`,代码从环境变量读。
3. **`.env` 必须加进 `.gitignore`,绝不提交到 git。** 提交一次就等于公开,即使后来删了,git 历史里还在。

创建 `.env` 文件:

```bash
# .env  ——  绝不提交到版本库
ANTHROPIC_API_KEY=sk-ant-你的真实密钥
OPENAI_API_KEY=sk-你的真实密钥
```

创建/确认 `.gitignore`:

```gitignore
# .gitignore
.env
.env.*
.venv/
node_modules/
```

> 建议同时提交一个 `.env.example`(只写键名、不写真实值),方便协作者知道需要哪些变量:
> ```
> ANTHROPIC_API_KEY=
> OPENAI_API_KEY=
> ```

## 第一个请求:Hello LLM

先来个最朴素的版本,确认能跑通。

#### TypeScript

```typescript
// hello.ts —— 用 pnpm tsx hello.ts 运行
import "dotenv/config"; // 自动加载 .env 到 process.env
import Anthropic from "@anthropic-ai/sdk";

// 不传 apiKey 时,SDK 默认读取 process.env.ANTHROPIC_API_KEY
const client = new Anthropic();

const response = await client.messages.create({
  model: "claude-opus-4-8", // 模型 ID 会变,以官方文档为准
  max_tokens: 1024,
  messages: [{ role: "user", content: "用一句话解释什么是 AI Agent。" }],
});

// 响应内容是一个块数组,先判断 type 再取 text
for (const block of response.content) {
  if (block.type === "text") console.log(block.text);
}
```

#### Python

```python
# hello.py —— 用 python hello.py 运行
from dotenv import load_dotenv
import anthropic

load_dotenv()  # 自动加载 .env 到 os.environ

# 不传 api_key 时,SDK 默认读取 os.environ["ANTHROPIC_API_KEY"]
client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-8",  # 模型 ID 会变,以官方文档为准
    max_tokens=1024,
    messages=[{"role": "user", "content": "用一句话解释什么是 AI Agent。"}],
)

# 响应内容是一个块列表,先判断 type 再取 text
for block in response.content:
    if block.type == "text":
        print(block.text)
```

跑通了,恭喜——你已经在用大模型了。Claude 的调用走的是 `POST /v1/messages` 接口,SDK 只是帮你封装了请求细节。

> **给前端的提醒:** 上面用了顶层 `await`(TS)和直接调用(Python),都是同步写法的样子,但底层是一次网络请求。生产代码里你会想加上超时、重试、错误处理——这些 SDK 大多内置了(默认会重试 429 和 5xx),我们在[工程篇](../03-工程篇/15-成本与性能优化.md)再细说。

## 搭一个最小的 chat() 抽象

直接到处写 `client.messages.create(...)` 有个问题:**和厂商绑死了**。想换成 OpenAI 或开源模型,就得改一大片代码。

本书的做法是:封装一个薄薄的 `chat()` 函数,**把"发一段对话、拿一段回复"这件事统一掉**。换厂商时,只改 `chat()` 内部,业务代码一行不动。这就是序言里说的"多模型抽象",从这里开始建立。

#### TypeScript

```typescript
// llm.ts —— 全书复用的最小统一抽象
import "dotenv/config";
import Anthropic from "@anthropic-ai/sdk";

// 一条对话消息:角色 + 文本内容(先从最简单的纯文本开始)
export interface ChatMessage {
  role: "user" | "assistant";
  content: string;
}

const anthropic = new Anthropic();
const MODEL_ID = "claude-opus-4-8"; // 集中管理模型 ID,换模型只改这一处

// 统一入口:给一组消息,返回模型回复的纯文本
export async function chat(messages: ChatMessage[]): Promise<string> {
  const response = await anthropic.messages.create({
    model: MODEL_ID,
    max_tokens: 1024,
    messages,
  });
  // 把所有 text 块拼起来返回
  return response.content
    .filter((b) => b.type === "text")
    .map((b) => (b as { text: string }).text)
    .join("");
}

// 用一下
const reply = await chat([{ role: "user", content: "你好,你是谁?" }]);
console.log(reply);
```

#### Python

```python
# llm.py —— 全书复用的最小统一抽象
from dotenv import load_dotenv
import anthropic

load_dotenv()

_client = anthropic.Anthropic()
MODEL_ID = "claude-opus-4-8"  # 集中管理模型 ID,换模型只改这一处


# 统一入口:给一组消息,返回模型回复的纯文本
# messages 形如 [{"role": "user", "content": "..."}]
def chat(messages: list[dict]) -> str:
    response = _client.messages.create(
        model=MODEL_ID,
        max_tokens=1024,
        messages=messages,
    )
    # 把所有 text 块拼起来返回
    return "".join(b.text for b in response.content if b.type == "text")


if __name__ == "__main__":
    reply = chat([{"role": "user", "content": "你好,你是谁?"}])
    print(reply)
```

现在,业务代码只跟 `chat()` 打交道。**为什么这层抽象重要?**

- 换模型厂商,只改 `chat()` 里面——比如把 Anthropic 换成 OpenAI 的 `client.chat.completions.create(...)`(注意 OpenAI 的消息格式和返回结构略有不同,以官方文档为准)。
- 后面要加流式、工具调用、重试、日志,都集中加在这一层。
- 业务逻辑和"具体用哪个模型"解耦,符合前端最熟悉的"抽象边界"思想。

> 这只是雏形,先从纯文本起步。随着学习深入,我们会逐步给它加上流式、多模型路由、工具调用等能力。但核心思想不变:**业务不直接碰厂商 SDK。**

## 可选:用 Ollama 在本地跑开源模型

不想花钱、想在本地试开源模型(如 Llama、Qwen)?用 [Ollama](https://ollama.com)。它在本地起一个服务,并提供**与 OpenAI 兼容**的接口,所以你可以用 OpenAI 的 SDK 直接连。

```bash
# 装好 Ollama 后,拉一个模型并运行
ollama pull qwen2.5        # 或 llama3.1 等
ollama run qwen2.5         # 起服务,默认监听 11434 端口
```

用 OpenAI SDK 连本地 Ollama(把 baseURL 指过去即可):

#### TypeScript

```typescript
import OpenAI from "openai";

// 指向本地 Ollama 的 OpenAI 兼容端点;本地服务不校验 key,填个占位符即可
const local = new OpenAI({
  baseURL: "http://localhost:11434/v1",
  apiKey: "ollama", // 占位,本地不验
});

const res = await local.chat.completions.create({
  model: "qwen2.5", // 你 ollama pull 下来的模型名
  messages: [{ role: "user", content: "你好" }],
});
console.log(res.choices[0].message.content);
```

#### Python

```python
from openai import OpenAI

# 指向本地 Ollama 的 OpenAI 兼容端点;本地服务不校验 key,填占位符即可
local = OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="ollama",  # 占位,本地不验
)

res = local.chat.completions.create(
    model="qwen2.5",
    messages=[{"role": "user", "content": "你好"}],
)
print(res.choices[0].message.content)
```

> 本地模型免费、隐私好、适合开发调试,但效果通常不如顶级闭源模型,且吃显存/内存。开发用本地、生产用云端,是常见组合。生产级部署(如 vLLM)我们在[工程篇 · 部署与生产化](../03-工程篇/17-部署与生产化.md)再讲。

## 常见坑 / 最佳实践

- **密钥没加载进来(最常见)。** 报"未提供 API key / authentication error"。检查:`.env` 文件是否在项目根目录?TS 是否 `import "dotenv/config"`、Python 是否调了 `load_dotenv()`?变量名是否拼对(`ANTHROPIC_API_KEY` 而不是 `ANTHROPIC_KEY`)?
- **Node 版本太低。** 报 `fetch is not defined` 或 ESM 相关错误。升级到 18+,推荐 20。
- **Python 没激活虚拟环境。** 报 `ModuleNotFoundError: No module named 'anthropic'`。先 `source .venv/bin/activate`,看到 `(.venv)` 再 `pip install` / 运行。
- **计费没开通。** 报 403、`billing` 或额度相关错误。去厂商控制台确认已绑卡/充值。免费注册不等于能调用 API。
- **公司网络 / 代理问题。** 国内或企业内网常需走代理才能连通 api.anthropic.com / api.openai.com。表现为连接超时。配置 `HTTPS_PROXY` 环境变量,或在 SDK 里配置自定义 HTTP 客户端/代理。
- **模型 ID 写错或过时。** 报 404 / not found。模型 ID 更新很快,**以官方文档为准**;本书写 `claude-opus-4-8` 是当前可用值,你照搬前最好确认一下。
- **把 `.env` 提交了。** 一旦提交,密钥即视为泄露——立刻去控制台**吊销并重新生成**密钥,而不只是删文件。养成第一步就把 `.env` 写进 `.gitignore` 的习惯。

## 本章小结

1. 装好 **Node.js 18+/20+** 和 **Python 3.10+** 双环境;TS 用 pnpm/npm,Python 用 venv+pip 或 uv。
2. API Key 用**环境变量 + `.env` + dotenv**管理,`.env` 进 `.gitignore`,**绝不硬编码、绝不提交**。
3. 安装官方 SDK:TS 的 `@anthropic-ai/sdk` / `openai`,Python 的 `anthropic` / `openai`。
4. 跑通了 "Hello LLM",并封装了全书复用的最小 **`chat()` 抽象**——业务不直接碰厂商 SDK,换模型只改一处。
5. 可选用 **Ollama** 在本地跑开源模型(OpenAI 兼容端点 `http://localhost:11434/v1`)。
6. 记住常见坑:密钥未加载、Node 版本、虚拟环境、计费未开通、代理。

## 练习题

1. **(易)** 在自己机器上跑通 TS 和 Python 两个版本的 "Hello LLM",截图记录返回结果。
2. **(易)** 故意把 `.env` 里的密钥删掉一行,跑一下,观察报什么错——记住这个错误长什么样,以后好认。
3. **(中)** 给 `chat()` 加一个 `system` 参数(系统提示词),让你能给模型设定角色。两种语言都改一下。(提示:Claude 是顶层 `system` 字段。)
4. **(中)** 装好 Ollama,拉一个开源模型,用 OpenAI SDK 连上并成功对话一次。对比它和云端模型的回答质量。
5. **(难)** 改造 `chat()`,让它支持一个 `provider` 参数(`"anthropic" | "openai"`),根据参数路由到不同厂商的 SDK,但对外保持同样的输入输出。这就是"多模型抽象"的核心,本书后续会反复用到。
