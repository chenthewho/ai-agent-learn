# 资源与工具清单

> 一份"按需取用"的清单：每类工具给**名称 + 一句话用途 + 适用场景**，让你知道什么时候该掏出哪一个。
>
> 两条原则贯穿全书，这里再强调一次：
> 1. **先懂原理再用框架。** 框架是替你写 Agent 循环、工具调用、记忆的"脚手架"，但出了问题你得知道它替你做了什么。
> 2. **保持厂商无关。** 代码尽量通过一层薄抽象（一个 `chat()`/`generate()` 函数）调模型，方便随时换厂商。
>
> ⚠️ 模型 ID 和价格变化很快，单独放在文末的[易变信息表](#易变信息表截至-2026-年年中)里，并醒目标注"以官方文档为准"。URL 容易失效，本清单一律只给名称和关键词，你按名搜索即可找到最新官网。

## 一、模型 / API

| 名称 | 一句话用途 | 适用场景 |
|---|---|---|
| **Anthropic Claude** | 本书默认主力模型，擅长长上下文、工具使用、Agent 任务 | 需要强推理、长上下文、稳定工具调用的生产 Agent。Python SDK `anthropic`，TS SDK `@anthropic-ai/sdk`，走 `POST /v1/messages` |
| **OpenAI** | 另一主流闭源模型厂商，生态成熟、社区资料多 | 想要广为人知的 API 形态、或已有 OpenAI 生态依赖时。SDK：Python `openai` / TS `openai` |
| **Qwen（通义千问）系列** | 开源中文友好模型，可本地或自托管 | 需要私有化部署、数据不出内网、或中文场景较多时 |
| **Llama 系列** | Meta 的开源模型，社区生态最大 | 自托管、二次微调、研究实验 |
| **Ollama** | 本地一键跑开源模型（Llama/Qwen 等），暴露 OpenAI 兼容端点 | 本地开发、离线试验、零成本练手。默认端点 `http://localhost:11434/v1` |
| **vLLM** | 高性能开源推理服务，OpenAI 兼容 | 把开源模型部署到生产、追求吞吐与并发时 |

> 选型直觉：**先用 Claude/OpenAI 把功能跑通**（迭代快），有私有化/成本/合规需求时再换到 Qwen/Llama（经 Ollama 本地或 vLLM 生产）。因为代码走薄抽象层，切换成本很低。详见[工程篇 · Agent 框架选型](../03-工程篇/10-agent框架选型.md)与[基础篇 · 大语言模型基础](../01-基础篇/02-大语言模型基础.md)。

## ⚠️ 易变信息表（截至 2026 年年中）

> **下面的模型 ID 与价格会变，请务必以各厂商官方文档为准。** 这里给出截至 2026 年年中的主流型号，方便你有个体感。价格单位：**美元 / 每百万（1M）token**，格式为「输入价 / 输出价」。

### Anthropic Claude（已核实）

直接使用下列模型 ID 字符串，**不要自己加日期后缀**（如 `-20251114` 这类是错的，会 404）。

| 模型 | 模型 ID | 上下文窗口 | 价格（输入 / 输出，每百万 token） | 定位 |
|---|---|---|---|---|
| Claude Fable 5 | `claude-fable-5` | 1M | $10 / $50 | 最强，最难的推理与长时程 Agent 任务 |
| Claude Opus 4.8 | `claude-opus-4-8` | 1M | $5 / $25 | **本书默认推荐**，强推理 + 长上下文 + Agent |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | 1M | $3 / $15 | 性价比之选，速度与智能的平衡，适合高量生产 |
| Claude Haiku 4.5 | `claude-haiku-4-5` | 200K | $1 / $5 | 最快最便宜，适合简单/高频/对延迟敏感的任务 |

Claude 关键使用要点（这些是当前已核实的写法）：
- **思考**：4.6+ 模型用 `thinking: {type: "adaptive"}`，**不要用旧的 `budget_tokens`**（新模型会报 400）。
- **努力程度**：`output_config: {effort: "low"|"medium"|"high"|"xhigh"|"max"}`。
- **结构化输出**：`output_config: {format: {type: "json_schema", schema: {...}}}`（旧的 `output_format` 已废弃）。
- **工具调用**：`tools` 里传 `{name, description, input_schema}`；响应 `stop_reason: "tool_use"` 时执行工具，把结果作为 `tool_result` 放进下一轮 `user` 消息。
- **提示缓存**：`cache_control: {type: "ephemeral"}`，前缀匹配，能大幅省钱。
- **不支持**：assistant 预填充（prefill）在 4.6+ 会报 400。
- 选型直觉：默认 `claude-opus-4-8`；高量生产降本用 `claude-sonnet-4-6`；简单高频任务用 `claude-haiku-4-5`；最难任务上 `claude-fable-5`。

### OpenAI（用常见名指代，以官方为准）

| 常见名 | 大致定位 | 备注 |
|---|---|---|
| `gpt-4.1` | 旗舰能力 | 具体型号与价格**以 OpenAI 官方为准** |
| `gpt-4o` | 多模态、性价比 | 同上 |
| `gpt-4o-mini` | 轻量、便宜、快 | 适合简单/高频任务，价格以官方为准 |

> OpenAI 的确切型号名与价格更新频繁，本表只给方向，**务必查官方 pricing 页**。Function calling 用 `tools` 参数 + `tool_calls`；结构化输出用 `response_format: {type: "json_schema", ...}` 或 SDK 的 `parse` 助手。

### 开源模型（给方向，自行查最新）

| 系列 | 方向 | 备注 |
|---|---|---|
| **Qwen 系列**（如 Qwen2.5 等） | 中文友好、参数档位齐全（从小到大） | 经 Ollama 本地跑或 vLLM 生产部署；具体型号看官方仓库 |
| **Llama 系列**（如 Llama 3.x 等） | 社区生态最大、可微调 | 同上；选参数量看你的显存与延迟预算 |

## 二、Agent / 应用框架

| 名称 | 一句话用途 | 适用场景 |
|---|---|---|
| **LangChain** | 老牌、组件最全的 LLM 应用框架（链、工具、记忆、RAG 全有） | 想要"什么都有"的起步框架；但抽象较重，注意别被它绑架 |
| **LangGraph** | LangChain 出品，把 Agent 建模成"状态图"，控制流清晰可控 | 需要复杂、可控、可循环的 Agent 流程（条件分支、人审、回退）时首选之一 |
| **LlamaIndex** | 专注 RAG/数据接入的框架，文档/索引/检索能力强 | 以"喂数据给模型"为核心的知识库类应用 |
| **Vercel AI SDK** | 前端友好的 TS SDK，流式输出、UI 集成、厂商无关做得最顺 | **前端转型者首选**：React/Vue/Next 里做流式聊天、工具调用 UI |
| **Mastra** | TS 原生的 Agent 框架（工作流、工具、记忆、评测一体） | 想用纯 TS 端到端做 Agent、又要工程化能力时 |
| **CrewAI** | 以"角色协作"为核心的多 Agent 框架（Python） | 快速搭多角色协作（如研究员+写作+审校）原型 |
| **AutoGen** | 微软的多 Agent 对话/协作框架（Python） | 多 Agent 对话、可编程协作模式的研究与实验 |
| **OpenAI Agents SDK** | OpenAI 官方的轻量 Agent SDK（含 handoff、guardrails 等原语） | 想要简洁、贴近 OpenAI 生态的 Agent 编排 |

> 选型直觉：**前端/全栈出身优先 Vercel AI SDK 或 Mastra（TS 原生）**；要复杂可控流程看 LangGraph；纯 RAG 看 LlamaIndex；多 Agent 原型看 CrewAI/AutoGen。不确定就先**手写一遍循环**，再决定要不要框架。详见[工程篇 · Agent 框架选型](../03-工程篇/10-agent框架选型.md)。

## 三、向量库 / Embedding

### 向量库

| 名称 | 一句话用途 | 适用场景 |
|---|---|---|
| **pgvector** | 给 PostgreSQL 加向量检索能力的扩展 | **前端/全栈最易接受**：已经在用 Postgres，加个扩展就能做 RAG，无需引入新中间件。本书推荐起步选择 |
| **Qdrant** | 高性能开源向量数据库，过滤+检索强 | 数据量上来、需要丰富元数据过滤时 |
| **Chroma** | 轻量、易上手的开源向量库 | 本地开发、快速试验、小规模项目 |
| **Milvus** | 面向大规模的分布式向量数据库 | 海量向量、需要横向扩展的生产场景 |
| **Pinecone** | 托管（SaaS）向量数据库，免运维 | 不想自己运维、想快速上生产时 |

### Embedding 模型

| 名称 | 一句话用途 | 适用场景 |
|---|---|---|
| **OpenAI `text-embedding-3-small` / `-large`** | 通用、效果好的闭源 embedding | 起步首选，small 便宜、large 更准（型号/价以官方为准） |
| **bge 系列** | 开源 embedding，中文表现好 | 私有化、中文知识库、想自托管时 |
| **gte 系列** | 另一组常用开源 embedding | 同上，可与 bge 对比择优 |

> 选型直觉：先用 OpenAI embedding 把 RAG 跑通，**有私有化需求再换 bge/gte**（开源、可本地）。向量库从 **pgvector** 起步最省心，规模/性能不够再迁 Qdrant/Milvus，或直接上托管的 Pinecone。Embedding 可换、向量库可迁，所以早期别过度纠结。详见[核心能力篇 · RAG 检索增强生成](../02-核心能力篇/08-rag检索增强生成.md)。

## 四、可观测 / 评测

| 名称 | 一句话用途 | 适用场景 |
|---|---|---|
| **Langfuse** | 开源的 LLM/Agent 可观测平台（trace、用量、成本、评测） | 想自托管、看清每一步 trace/span 与花费时的首选 |
| **LangSmith** | LangChain 出品的可观测+评测平台 | 用 LangChain/LangGraph 时配套体验最顺 |
| **Helicone** | 以代理网关方式做用量/成本/缓存观测 | 想"加一层代理"就拿到调用日志与成本统计时 |
| **Phoenix / Arize** | 开源（Phoenix）+ 商业（Arize）的 LLM 观测与评测 | 偏数据科学视角的追踪、评测、漂移分析 |
| **promptfoo** | 提示词/模型的对比测试与评测工具 | 系统比较不同提示/模型/参数的效果（含 LLM-as-Judge） |
| **Ragas** | 专门评测 RAG 系统质量的框架 | 单独衡量检索质量、答案忠实度、相关性 |
| **OpenTelemetry** | 通用可观测标准（trace/metrics/logs） | 想把 Agent 观测接入公司既有的标准化监控体系 |

> 选型直觉：开发期接 **Langfuse**（自托管、看 trace 直观）；评测期用 **promptfoo**（通用对比）+ **Ragas**（专评 RAG）；要纳入公司统一监控走 **OpenTelemetry**。评测要分层：RAG 单独评检索、Agent 单独评轨迹（Trajectory）。详见[工程篇 · 评测与测试](../03-工程篇/13-评测与测试.md)与[工程篇 · 可观测性与调试](../03-工程篇/14-可观测性与调试.md)。

## 五、MCP 生态

| 名称 | 一句话用途 | 适用场景 |
|---|---|---|
| **MCP 官方 SDK** | 实现 MCP 协议的官方库（多语言），用来写/连 MCP Server | 想给 Agent 接标准化外部工具/数据源时 |
| **文件系统类 MCP Server** | 让 Agent 读写本地/指定目录文件 | 给 Agent 安全可控的文件读写能力 |
| **Git / GitHub 类 MCP Server** | 让 Agent 操作仓库、读 PR、提交代码 | 代码类 Agent、自动化研发流程 |
| **数据库类 MCP Server** | 让 Agent 查询数据库 | 让 Agent 安全地读你的业务数据做问答/分析 |
| **搜索 / 网页抓取类 MCP Server** | 让 Agent 联网搜索、抓取网页 | 需要最新信息、超出模型训练截止时间的任务 |
| **第三方 SaaS 类 MCP Server**（如项目管理、协作工具等） | 让 Agent 接入企业常用 SaaS | 把 Agent 嵌入现有工作流（建任务、发消息等） |

> MCP 像"工具界的统一插头"：写一次 Server，多个 Agent/客户端都能用。关键词搜 "MCP server" + 你要接的系统名，一般都能找到现成实现。**注意鉴权：凭证别塞进提示词，走专门的凭证通道。** 详见[工程篇 · MCP 与工具生态](../03-工程篇/11-mcp与工具生态.md)。

## 六、学习资源

| 名称 / 方向 | 一句话用途 | 适用场景 |
|---|---|---|
| **Anthropic 官方文档（platform.claude.com / docs）** | Claude API、工具使用、提示缓存、Agent 等权威说明 | 写 Claude 代码时的第一手资料，**模型/参数以它为准** |
| **OpenAI 官方文档** | OpenAI API、function calling、结构化输出等 | 用 OpenAI 时的权威资料 |
| **"Building effective agents"（Anthropic 工程博客）** | 讲清"什么时候该上 Agent、怎么设计简单可靠的 Agent" | 设计 Agent 前必读的方法论，避免过度工程 |
| **ReAct 论文** | Agent 经典范式（推理+行动交替）的出处 | 想吃透 Agent 循环底层思路时 |
| **Reflexion / Toolformer 等论文** | 反思、工具使用等关键能力的代表性研究 | 深入理解反思、工具调用的来龙去脉 |
| **RAG 原始论文** | 检索增强生成的奠基工作 | 想从根上理解 RAG 为什么有效时 |
| **Attention Is All You Need** | Transformer/注意力机制奠基论文 | 想理解大模型底层结构时（够用即可，不必逐字啃） |
| **arXiv / Papers with Code** | 追前沿论文与对应实现 | 持续跟进新方法、找可复现代码 |
| **各框架官方文档与 GitHub** | LangGraph/Mastra/Vercel AI SDK 等的权威用法与源码 | 学用法、读源码、看 examples 时 |
| **MCP 官方文档与协议规范** | MCP 协议、SDK、Server 列表 | 接入或开发 MCP 工具时 |
| **技术社区**（开发者论坛、框架的 Discord/GitHub Discussions、相关公众号/Newsletter） | 跟同行交流、追踪变更、解决踩坑 | 遇到问题求助、追前沿动态 |

> 读法建议：**官方文档**当工具书随用随查（尤其模型/参数细节）；**经典论文**先读摘要和图、能复述核心思路即可；**框架源码**精读 1–2 个的 Agent 循环实现，理论落地的细节都在那。详见各章末尾的"延伸阅读"。

## 一图速记：从需求到上线，每步掏哪个工具

```
需求来了
  │
  ├─ 选模型 ──────────▶ Claude（默认）/ OpenAI / 开源(Qwen·Llama via Ollama·vLLM)
  │
  ├─ 搭 Agent ────────▶ 先手写循环 → 再选框架(Vercel AI SDK / Mastra / LangGraph ...)
  │
  ├─ 接知识 (RAG) ────▶ Embedding(OpenAI / bge·gte) + 向量库(pgvector 起步 → Qdrant/Milvus/Pinecone)
  │
  ├─ 接工具 ──────────▶ 自写工具 / MCP Server(文件·Git·DB·搜索·SaaS)
  │
  ├─ 看清楚 ──────────▶ 可观测(Langfuse / OpenTelemetry) + 评测(promptfoo / Ragas)
  │
  └─ 上线 ────────────▶ 提示缓存省钱 + 限流/重试 + 安全护栏 + 部署
```

> 工具是手段不是目的。**先把最小闭环跑通，再按瓶颈逐项升级。** 不确定某个概念，回[术语表](./01-术语表.md)；不确定学习节奏，看[学习路线图](./02-学习路线图.md)。
