# 学习路线图

> 三套节奏，按你的时间和目标挑一套。每周都明确给出**读哪些章节 + 动手做什么 + 产出什么**——产出比"读完"重要得多，简历和面试看的都是产出。
>
> 章节路径都用相对链接，点了直接跳。前置约定：你已经会 TS/前端那套（见[读者画像与前言](../00-前言/00-序言与导读.md)），Python 看得懂基础即可。

## 怎么选

| 你的情况 | 选这套 |
|---|---|
| 时间紧，想最快做出一个能跑、能演示的 Agent | **4 周快速上手** |
| 想系统过一遍、能独立做项目、准备求职 | **8 周完整体系** |
| 想吃透工程细节、能在团队里扛起 Agent 方向、冲面试 | **12 周深度精通** |

> 三套是递进关系：12 周 = 8 周 + 加深；8 周 = 4 周 + 补全。中途想升级随时切。

## 前端转型者专属建议

你已经有的优势别浪费，缺的短板要补对地方。

**1. TS 与 Python 的精力怎么分？**
- **主力用 TS**：全栈集成、流式前端、Vercel AI SDK / Mastra 这条线天然是你的主场，先在这上面把 Agent 跑顺，建立信心。
- **Python 够用就行**：看得懂、能改、能跑示例即可。它在数据处理、向量库脚本、很多论文/开源 demo 里是默认语言，**你不需要写得多漂亮，但要能读懂别人的 Python 实现并迁移到 TS**。本书每个核心概念都给了双语代码，正好对照。
- 建议比例：约 **70% TS / 30% Python**。遇到只有 Python 版的好库（如某些向量/评测工具），用 Python 写个薄薄的服务，再用你熟的 HTTP/SSE 接给 TS 前端——这恰恰是你的强项。

**2. 把已有技能直接映射过来**（少走弯路）：
- 异步/Promise/事件循环 → 直接理解 Agent 循环、并行工具调用（`Promise.all`）。
- fetch/SSE/WebSocket → 直接上手流式输出与前端集成。
- 组件化/状态管理 → 直接理解短期/长期记忆、上下文工程。
- npm 生态/打包 → 直接理解框架选型、依赖与部署。
- 表单校验/类型 → 直接理解 JSON Schema、结构化输出、护栏。

**3. 怎么把项目变成简历项目？**
- **写清"为什么"而非"是什么"**：简历别写"用了 LangGraph 做了个 Agent"，要写"为降低幻觉用 RAG + 重排，把错误率从 X 降到 Y；为控成本做提示缓存 + 模型分级，token 成本降 Z%"。带数字、带取舍。
- **每个项目准备一段 2 分钟讲解**：解决什么问题、核心难点、你怎么取舍、踩了什么坑、怎么验证效果（评测）。这正是[面试篇 · 项目经验与开放题](../05-面试篇/04-项目经验与开放题.md)练的。
- **留下证据**：GitHub 仓库（README 要能让人 5 分钟跑起来）、一段演示录屏、一张架构图（ASCII/Mermaid 都行）。
- **强调全栈落地**：你能从模型一路打通到生产 UI——这是很多纯算法背景的人不具备的差异化优势，[项目四 全栈 AI Agent 产品](../04-实战篇/项目4-全栈ai-agent产品.md)就是用来展示这一点的。

**4. 别陷入的两个坑**：
- 别一上来就堆框架。**先手写一遍 Agent 循环**，你才知道框架替你做了什么、出问题时去哪查。
- 别想着先把机器学习/数学补完再开始。本书把向量、注意力等都讲到"够用"的程度，**边做边补**比啃理论高效得多。

## 学完之后，怎么持续进阶

这个领域变化快，"学完"只是起点。保持手感的几条路：

- **读论文（挑经典 + 追前沿）**：先吃透几篇地基论文方向——ReAct、Reflexion、Toolformer、RAG 原始论文、注意力机制（Attention Is All You Need）。再用 arXiv、Papers with Code 追新。读法：先读摘要和图，能复述核心思路即可，别逐字啃。
- **读框架源码**：精读 1–2 个主流框架的 Agent 循环与工具调度实现（LangGraph、Mastra、OpenAI Agents SDK 等，见[资源与工具清单](./03-资源与工具清单.md)）。源码会告诉你"理论落地时真正的细节在哪"。
- **跟官方文档与变更**：模型 API 变化快（见[易变信息表](./03-资源与工具清单.md#易变信息表截至-2026-年年中)）。订阅 Anthropic / OpenAI 的文档更新与 changelog，关注新模型、新参数、新工具能力。
- **参与开源**：从给框架/MCP Server 提 issue、补文档、修小 bug 开始，逐步贡献功能。既练手又攒履历。
- **持续做小项目**：每出一个新能力（新工具、新协议、新模型特性）就做个小 demo 试一遍——这是你前端时代就验证过的最佳学习方式。
- **关注前沿方向**：Agentic RAG、长时程自主 Agent、多 Agent 协作、MCP 工具生态、评测与可观测性正在快速演进，挑你最感兴趣的一条深扎。

> 路线图是地图不是铁律。卡住了就回到对应章节的"常见坑 / 最佳实践"，或去[术语表](./01-术语表.md)对一下概念。动手 > 阅读，产出 > 收藏。
